Hello, 

You can compile Pink on Windows under Python 2.6, there should be no problem. 
The CMakeLists.txt file however is neither prepared for mingw nor for any 
other tool. In my unfortunate experience CMake does not work on write once 
configure anywhere basis. 

Pink can be compiled natively on Windows using the Visual Studio compiler. The 
build automation tool on Windows is bjam (boost-jam). It is used, because I 
could not link the boost-python shared library with CMake on Windows. Someone 
who knows the Visual Studio compiler better could make a CMakeLists.txt file. 

There is also a setup.py file which generates the installer. The scripts are 
not yet linked together so the building is rather manual. 

For building Pink on Windows, You need the boost libraries and some knowledge 
of the command line. Also referring to Your future emails I will make a 
detailed howto tomorrow. 

There is no need to write Your jamroot  or setup.py file, I have already 
prepared both. They should be in the source package. Anyway I will try to put 
it together tomorrow.
