Dear All,

My name is Kacper Pluta and I'm the master student from Poland. Since 2011 I 
have been working with Ph.D Janaszewski and M.Sc. Postolski. We use Pink and 
libpink very widely and I just need to say that: Pink is great and thank you 
for your work and sharing it as open source. 

Unfortunately we have been using too many versions of Pink, sometimes with our 
patches, and some time ago I have come to the conclusion that the only way out 
of this situation is to support the main source of pink and drop all of the 
old versions.

Today I want to say "Hello" and send you only few things but as I mentioned I 
want to support your work.

List of content( https://www.wuala.com/hilsim/Pink-070713/?key=r7tLZWtAhMTP ):

Patches:

-- Patch for raw2pgm(file raw2pgm.c.patch):
---- short data type support now work properly

-- Patches for ltangent.cxx(file ltangent.cxx.patch):
---- wrong memory releases,
---- memory leaks(debugged with Valgrind),
---- nan values in output of lambdaMST2/3D (debugged with C99 function 
feenableexcept(FE_DIVBYZERO | FE_INVALID | FE_OVERFLOW)) 

New things:

-- Bridge between libpink and ITK(files ImageTypeConverter.h, 
ImageTypeConverter.hxx)

-- Adaptive medial axis described in M. Postolski Ph.D. thesis (file: 
adaptivemedialaxis.c) I also added files of airway tree to compare this method 
with lambdamedialaxis(files; medial.png(threshold: 2.5), 
adaptive.png(threshold: 0.65), tree.png) 

-- changevalue.c - swap value in input file to another(equivalent may already 
exist).
 
Finally, I would like to ask about the test files. For example in file 
lambdamedialaxis.c there are references to file: 

%IMAGES/2dbyte/binary/b2hebreu.pgm

but in pink dir "images" it not exist.
