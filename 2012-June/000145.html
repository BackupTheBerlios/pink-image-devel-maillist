Hello all,

So given the previous email, I think it would require lots of efforts to duplicate numpy-like arithmetics under pink. Yet this kind of arithmetics is extremely useful. This is only one example, there are many other toolkits that are image-related under python, and almost without exception they use numpy as their basis.

Given the current work that Laszlo is doing for pink <-> numpy conversion, how much effort, or overhead, would it be for *all* pink function to accept a numpy array as imput instead of our current "proprietary" image format ? Also, if a pink image is given only numpy arrays as input, can it transparently output a numpy array instead of a pink image ?

This would allow us to leverage all the enormous effort put into numpy/scipy/scikit by other groups, and would likely make pink much much more popular and useful.

What do people think ?
