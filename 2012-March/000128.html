Salut Edward,

In general please try to send email to the developers' list so that several people can see it.

There are several solutions

1- in fact Pink can read TIFF images.


import pink
a = pink.cpp.readimage('toto.tif') 


should work fine. Since TIFF is not really specified for 3D, I'm not sure what will happen.... 

2- You can read many formats with imview: 

http://hugues.zahlt.info/Imview.html

Under Ubuntu and other Debian-related distributions, imview can be installed by apt-get , synaptic, etc.

Then imview can export in PGM format.


3- the extended PGM images consiste of a very simple header followed by the data. The documentation of the format in the source of pink is under

doc/format.txt

It may be a little cryptic to read...

So writing a header by hand before the data is doable.

4- you can use the function 

pink.cpp.read_raw_image()


This should be the simplest option, but I don't know how to use it. The help string is not as helpful as it could be:

>
