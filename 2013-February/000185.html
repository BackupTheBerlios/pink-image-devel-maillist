Hello,

I am writing about the merging. I was looking for some tutorial, but the
simplest would be to tell You my experiences.

You can start the merging with
http://mercurial.selenic.com/wiki/TutorialMerge

this gives You an introduction as to how do You need to merge the branches.
Before the merge You can clone a clean repository, so if anything is messed
up You can just clone a new one. You always do two-way merge first
default->personal then personal->default.
All the commands are local so

hg checkout personal
hg merge default

will do the default->personal merge.

If there is a conflict You need a merge tool; for mercurial I use kdiff3
which You can setup as:

[ui]
merge    = kdiff3

[extensions]
hgext.extdiff =

[extdiff]
cmd.kdiff3 =

[merge-tools]
kdiff3.args = $base $local $other -o $output

in the ~/.hgrc config file. You can also use meld (git's merging tool),
both are good. After You have reviewed Your changes You have to merge the
changes manually and push them back to the main repository.

Please write me questions if any arise. I also suggest to look into hg
shelve for those changes, which You do not want to put into the repository
(like debug information).

Laszlo
