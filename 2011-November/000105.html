Hi Laszlo,

I'm trying to compile r669 on OSX 10.6.8 (in order to access the
powerwatershed).  I have previously compiled r596 on this machine without
problems (with the python frontend on).

It seems that GSL is now a requirement, and I have this installed with
 headers in /opt/local/include/gsl/*.h

When running ccmake, a config error occurs stating that:

 CMake Error at CMakeLists.txt:213 (message):
   gsl header file failed to compile.  If it is installed, than check the
gsl
   flags

Looking at CMakeLists it seems to be a failing search for gsl_interp.h i.e.

  CHECK_INCLUDE_FILE( gsl/gsl_interp.h HTEST )
  if (NOT HTEST)
     message( FATAL_ERROR "gsl header file failed to compile. If it is
installed, than check the gsl flags")
         endif(NOT HTEST)


There definitely is the file "/opt/local/include/gsl/gsl_interp.h" on my
system, but it is not being found by ccmake.

If I ignore this error by commenting out the check in CMakeLists, I get an
error:
 /System/Library/Frameworks/Python.framework/Headers/pyconfig.h

 CMake Error at CMakeLists.txt:358 (message):
    pyconfig.h header file failed to compile. If it is installed, than
check the python flags and directories

The pyconfig.h file is definitely on my system.

In summary, the build process isn't finding files that I know really do
exist on my system.  Any thoughts?  Is there a new hard-coded search path
that needs to be changed?

Mike.
