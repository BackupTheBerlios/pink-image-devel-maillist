As pointed out earlier Pink suffers from long compilation times. This is 
caused partially by the extensive template usage (both on Pink's and on 
Boost's sides) and partially by the fact, that most of the Python wrapper is 
accumulated in a single file. 

To speed up the compilation time, I am envisaging to break up pypink.cpp into 
several subfiles. This way, the compilation can be speed up (hopefully) two 
ways. Parallel compilation is already supported by Pink using "make -j16" 
where 16 is the number of threads. So if separating pypink, the separate files 
will be compiled in parallel.  Secondly if a function is wrapped to Python, 
then only the specific file containing the function has to be recompiled, 
speeding up thus the development time.
