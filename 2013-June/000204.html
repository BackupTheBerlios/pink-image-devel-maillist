> > For future reference, please note that the pink-svn CMake build system

> > (mercurial) repository.

> > Here is how can You build pink:
> > 
> > sudo apt-get install gcc g++ cmake cmake-qt-gui libx11-dev make
> > mercurial
> > ncurses-dev libgsl0-dev liblapack-dev libboost1.53-all-dev libtiff4-dev
> > cmake-
> > curses-gui python-matplotlib python-vtk ipython python-numpy
> > python-imaging python-imaging-tk
> > 
> > hg clone https://pinkhq.com/hg/pink
> > 
> > mkdir build
> > cd build
> > 
> > cmake ../pink/ -DPYTHON_FRONT_END=ON -DCMAKE_VERBOSE_MAKEFILE=ON
> > cmake ../pink/ -DPYTHON_FRONT_END=ON -DCMAKE_VERBOSE_MAKEFILE=ON
> > cmake ../pink/ -DPYTHON_FRONT_END=ON -DCMAKE_VERBOSE_MAKEFILE=ON
> > make
> > 
> > note that CMake is an iterative process, that's why we configure the
> > directory
> > three times. You can change further settings with
> > cmake-gui .
> > 
> > You can start using pink by
> > ./pink.sh
> > 
> > and then
> > 
> > import pink
