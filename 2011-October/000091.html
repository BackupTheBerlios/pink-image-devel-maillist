Hello,

pink.cpp.heightmaxima only work on char images but does not say so

using cells.pgm from trunk/tutorial/python/HT-TP1/images/cells.pgm

try

im=pink.cpp.readimage("cells.pgm")
dist=pink.cpp.dist(im,1) # or 5
height=pink.cpp.heightmaxima(dist,4,4)

Only the top 1/4 of the image is processed, and it is incorrect.

However, 

1- Using height maxima on distance functions is an important trick for filtering ultimate erosions. Is there another way to do it?
2- some objects can have a radius > 255 , so it is important to be able to run height maxima on int (or float) images
3- if a function does not work on a data type, it should reject it, not work incorrectly and silently.

Moreover:

- pink.cpp.segmentheight has the same problem
- The command line functions have the same problem


We should get heightmaxima to work at least on int image, if not, at least say that it doesn't work.
