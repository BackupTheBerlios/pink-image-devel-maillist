Hi,

I work on pink, I am an intern at ESIEE. I tried some experimenting with
Doxygen and it seems that one can insert/embedd
image<http://www.star.bnl.gov/public/comp/sofi/doxygen/commands.html#cmdimage>s
within the documentation. I have inserted the respective links. This
simplifies our case if it works for python codes. I have tested this on  a
sample C code for now.

For the python examples I guess it would be good to explore the doxypy
<http://code.foosel.org/doxypy>filter.

Nevertheless if this should work - it would reduce the requirement for
a *script
*within pink(still to be decided) to just running doxygen after producing
the image and other outputs. One should keep in mind that the outputs to be
generated are to be uniquely named. This might be a minor problem if the the
tool doesnt allow recursive searching. I shall test this within pink
shortly.

The purpose is to produce documentation that has the usual doxygen level for
all the basic operators and functions in pink, while including the input and
outputs at the application and operator level. This way an user can write
his own example showing the different steps and outputs produced.
