This solution may be feasible. In fact all we need is a pre-parser, which 
takes the python script parts, executes them, and replaces the image links 
where appropriate in the doxygen file. 

The code can look like this:

my example code:

import pink.cpp as pink
image1 = pink.readimage("path/relative/to/svn/or/hg")
image2 = pink.some_function(image1, 3)

in the end the pre-parser would add two lines to the script (before 
executing):

image1.writeimage("/internal/image/name/for/the/script")
image2.writeimage("/internal/image/name/for/the/script2")

and then in the rest of the code it would replace $image1 and $image2 with the 
relative paths of the images.

I would also need to look into the proposed self-testing feature, because, we 
maybe able to unify the two.

The only question is how  easy it would be to use?

Have you find other tools, independent from doxygen, for documentation auto-
generation or testing?
