Yes this is intentional. You cannot contaminate the namespace with executables 
such as convert, min or max. There are two solutions. One is the pink.command 
format the other would be to create an executable called pink, which would 
invoke the individual bash commands in the form

pink command param_1 param_2 param_3 ... param_n

The latter solution has been applied in projects such as svn, git, mercurial, 
but the pink.command solution seemed to be more conservative.
