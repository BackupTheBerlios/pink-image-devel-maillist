Dear Pink Users, 

After much work I have managed to change Pink's CMake file to also compile on Windows(tm).

I have created packages with an installer for Python 2.6 64bit and Python 2.7 64bit. These packages a continuously built after each commit in the repository together with the Linux packages. The packages are full and contain all the standart and development operators. 

https://www.pinkhq.com/download/packages/pink/windows/

What is missing: The Python 3.2 package does not compile yet, and I will only investigate it later. The imview function is not yet ready as it is using named pipes which are not present on Windows(tm). The unit tests are not applied. 

Now the compilation is much more difficult than on Linux. It includes the need to compile all the dependencies by hand. For those who would like to try it all the config files and scripts used to build the packages are in the repository at pink/data/configs/*.


Cordially, 

Laszlo Marak
